MaterialComponent

MaterialComponent

Abstract base class for materials.

Materials describe the appearance of [page:Object objects].
They are defined in a (mostly) renderer-independent way, so you don't have to
rewrite materials if you decide to use a different renderer.

The following properties and methods are inherited by all other material types
(although they may have different defaults).

[property:any alphaMap]Input Public

The alpha map is a grayscale texture that controls the opacity across the surface (black: fully transparent; white: fully opaque). Default is null.
Only the color of the texture is used, ignoring the alpha channel if one exists.
For RGB and RGBA textures, the [page:WebGLRenderer WebGL] renderer will use the green channel when sampling this texture due to the extra bit of precision provided for green in DXT-compressed and uncompressed RGB 565 formats. Luminance-only and luminance/alpha textures will also still work as expected.

[property:any aoMap]Input Public

The red channel of this texture is used as the ambient occlusion map. Default is null.
The aoMap requires a second set of UVs.

[property:number aoMapIntensity]Input Public

Intensity of the ambient occlusion effect. Default is 1. Zero is no occlusion effect.

[property:any bumpMap]Input Public

The texture to create a bump map. The black and white values map to the perceived depth in relation to the lights.
Bump doesn't actually affect the geometry of the object, only the lighting. If a normal map is defined this will be ignored.

[property:number bumpScale]Input Public

How much the bump map affects the material. Typical ranges are 0-1. Default is 1.

[property:number clearcoat]Input Public

Represents the intensity of the clear coat layer, from *0.0* to *1.0*.
Use clear coat related properties to enable multilayer materials that have a thin translucent layer over the base layer. Default is *0.0*.

[property:any clearcoatNormalMap]Input Public

Can be used to enable independent normals for the clear coat layer. Default is *null*.

[property:number clearcoatNormalScale]Input Public

How much [page:.clearcoatNormalMap] affects the clear coat layer, from *(0,0)* to *(1,1)*. Default is *(1,1)*.

[property:number clearcoatNormalScaleX]Input Public

How much [page:.clearcoatNormalMap] affects the clear coat layer, from *(0,0)* to *(1,1)*. Default is *(1,1)*.

[property:number clearcoatNormalScaleY]Input Public

How much [page:.clearcoatNormalMap] affects the clear coat layer, from *(0,0)* to *(1,1)*. Default is *(1,1)*.

[property:number clearcoatRoughness]Input Public

Roughness of the clear coat layer, from *0.0* to *1.0*. Default is *0.0*.

[property:boolean clipping]Input Public

Defines whether this material supports clipping; true to let the renderer pass the clippingPlanes uniform. Default is false.

[property:ThreeColor color]Input Public

Color of the material, by default set to white (0xffffff).

[property:number colorMultiply]Input Public

Color of the material multiply (1)

[property:string combine]Input Public

How to combine the result of the surface's color with the environment map, if any.
Options are [page:Materials THREE.Multiply] (default), [page:Materials THREE.MixOperation], [page:Materials THREE.AddOperation]. If mix is chosen, the [page:.reflectivity] is used to blend between the two colors.

Color of the material multiply (1)

[property:any diffuseMap]Input Public

The red channel of this texture is used as the ambient occlusion map. Default is null.
The aoMap requires a second set of UVs.

[property:number displacementBias]Input Public

The offset of the displacement map's values on the mesh's vertices.
Without a displacement map set, this value is not applied. Default is 0.

[property:any displacementMap]Input Public

The displacement map affects the position of the mesh's vertices. Unlike other maps which only affect the light and shade of the material the displaced vertices can cast shadows, block other objects, and otherwise act as real geometry. The displacement texture is an image where the value of each pixel (white being the highest) is mapped against, and repositions, the vertices of the mesh.

[property:number displacementScale]Input Public

How much the displacement map affects the mesh (where black is no displacement, and white is maximum displacement). Without a displacement map set, this value is not applied.
Default is 1.

[property:ThreeColor emissive]Input Public

Emissive (light) color of the material, essentially a solid color unaffected by other lighting.
Default is black.

[property:number emissiveIntensity]Input Public

Intensity of the emissive light. Modulates the emissive color. Default is 1.

[property:number emissiveMultiply]Input Public

Emissive (light) color multiply of the material, essentially a solid color unaffected by other lighting.

[property:any envMap]Input Public

The environment map. Default is null.

[property:number envMapIntensity]Input Public

Scales the effect of the environment map by multiplying its color.

Encoding for depth packing. Default is [page:Textures BasicDepthPacking].

Fragment shader GLSL code. This is the actual code for the shader. In the example above, the *vertexShader* and *fragmentShader* code is extracted from the DOM; it could be passed as a string directly or loaded via AJAX instead.

[property:number gapSize]Input Public

The size of the gap. Default is *1*.

[property:string glslVersion]Input Public

Defines the GLSL version of custom shader code. Only relevant for WebGL 2 in order to define whether to specify GLSL 3.0 or not. Valid values are *THREE.GLSL1* or *THREE.GLSL3*. Default is *null*.

[property:any lightMap]Input Public

The light map. Default is null. The lightMap requires a second set of UVs.

[property:number lightMapIntensity]Input Public

Intensity of the baked light. Default is 1.

[property:boolean lights]Input Public

Defines whether this material uses lighting; true to pass uniform data related to lighting to this shader. Default is false.

[property:string linecap]Input Public

Define appearance of line ends. Possible values are 'butt', 'round' and 'square'.
Default is 'round'.
This corresponds to the [link:https://developer.mozilla.org/en/docs/Web/API/CanvasRenderingContext2D/lineCap 2D Canvas lineCap] property and it is ignored by the [page:WebGLRenderer WebGL] renderer.

Notice - case insensitive.

[property:string linejoin]Input Public

Define appearance of line joints. Possible values are 'round', 'bevel' and 'miter'. Default is 'round'.
This corresponds to the [link:https://developer.mozilla.org/en/docs/Web/API/CanvasRenderingContext2D/lineJoin 2D Canvas lineJoin] property and it is ignored by the [page:WebGLRenderer WebGL] renderer.

Notice - case insensitive.

[property:number linewidth]Input Public

Controls line thickness. Default is *1*.
Due to limitations of the [link:https://www.khronos.org/registry/OpenGL/specs/gl/glspec46.core.pdf OpenGL Core Profile] with the [page:WebGLRenderer WebGL] renderer on most platforms linewidth will always be 1 regardless of the set value.

[property:any map]Input Public

The color map. Default is null.

[property:any matcap]Input Public

The matcap map. Default is null.

[property:number metalness]Input Public

How much the material is like a metal. Non-metallic materials such as wood or stone use 0.0, metallic use 1.0, with nothing (usually) in between. Default is 0.0. A value between 0.0 and 1.0 could be used for a rusty metal look. If metalnessMap is also provided, both values are multiplied.

[property:boolean morphNormals]Input Public

Defines whether the material uses morphNormals. Set as true to pass morphNormal attributes from the geometry to the shader. Default is *false*.

[property:boolean morphTargets]Input Public

Define whether the material uses morphTargets. Default is false.

[property:number nearDistance]Input Public

The near value of the point light's internal shadow camera.

[property:any normalMap]Input Public

The texture to create a normal map. The RGB values affect the surface normal for each pixel fragment and change the way the color is lit. Normal maps do not change the actual shape of the surface, only the lighting.
In case the material has a normal map authored using the left handed convention, the y component of normalScale should be negated to compensate for the different handedness.

[property:string normalMapType]Input Public

The type of normal map.
Options are [page:constant THREE.TangentSpaceNormalMap] (default), and [page:constant THREE.ObjectSpaceNormalMap].

Notice - case insensitive.

[property:number normalScale]Input Public

How much the normal map affects the material. Typical ranges are 0-1.
Default is a [page:Vector2] set to (1,1) *1*.

[property:number normalScaleX]Input Public

How much the normal map affects the material. Typical ranges are 0-1.
Default is a [page:Vector2] set to (1,1).

[property:number normalScaleY]Input Public

How much the normal map affects the material. Typical ranges are 0-1.
Default is a [page:Vector2] set to (1,1).

[property:number referencePositionX]Input Public

The position of the point light in world space.

[property:number referencePositionY]Input Public

The position of the point light in world space.

[property:number referencePositionZ]Input Public

The position of the point light in world space.

[property:number reflectivity]Input Public

How much the environment map affects the surface; also see [page:.combine].
The default value is 1 and the valid range is between 0 (no reflections) and 1 (full reflections).

[property:any reflector]Input Public

The reflector of material component

[property:number refractionRatio]Input Public

The index of refraction (IOR) of air (approximately 1) divided by the index of refraction of the material.
It is used with environment mapping modes [page:Textures THREE.CubeRefractionMapping] and [page:Textures THREE.EquirectangularRefractionMapping].
The refraction ratio should not exceed 1. Default is *0.98*.

[property:number resolutionX]Input Public

The resolutionX of material component

[property:number resolutionY]Input Public

The resolutionY of material component

[property:number rotation]Input Public

The rotation of the sprite in radians. Default is 0.

[property:number roughness]Input Public

How rough the material appears. 0.0 means a smooth mirror reflection, 1.0 means fully diffuse. Default is 1.0.
If roughnessMap is also provided, both values are multiplied.

[property:any roughnessMap]Input Public

The green channel of this texture is used to alter the roughness of the material.

[property:number scale]Input Public

The scale of the dashed part of a line. Default is *1*.

[property:string|ShaderType shader]Input Public

The shader type

[property:ThreeColor sheen]Input Public

If a color is assigned to this property, the material will use a special sheen BRDF intended for rendering cloth materials such as velvet.
The sheen color provides the ability to create two-tone specular materials. *null* by default.

[property:number sheenMultiply]Input Public

The sheenMultiply of material component

[property:number shininess]Input Public

How shiny the [page:.specular] highlight is; a higher value gives a sharper highlight. Default is *30*.

[property:number size]Input Public

[property:ThreeColor specular]Input Public

Specular color of the material. Default is a [page:Color] set to *0x111111* (very dark grey).

[property:number thickness]Input Public

How much the material is like a metal. Non-metallic materials such as wood or stone use 0.0, metallic use 1.0, with nothing (usually) in between. Default is 0.0. A value between 0.0 and 1.0 could be used for a rusty metal look. If metalnessMap is also provided, both values are multiplied.

[property:number transmission]Input Public

Degree of transmission (or optical transparency), from *0.0* to *1.0*. Default is *0.0*.
Thin, transparent or semitransparent, plastic or glass materials remain largely reflective even if they are fully transmissive.
The transmission property can be used to model these materials.
When transmission is non-zero, [page:Material.opacity opacity] should be set to *1*.

[property:ThreeUniforms uniforms]Input Public

An object of the form:
json
{ "uniform1": { value: 1.0 }, "uniform2": { value: 2 } }

specifying the uniforms to be passed to the shader code; keys are uniform names, values are definitions of the form
json
{ value: 1.0 }

where *value* is the value of the uniform. Names must match the name of the uniform, as defined in the GLSL code. Note that uniforms are refreshed on every frame, so updating the value of the uniform will immediately update the value available to the GLSL code.

[property:string vertexShader]Input Public

Vertex shader GLSL code. This is the actual code for the shader. In the example above, the *vertexShader* and *fragmentShader* code is extracted from the DOM; it could be passed as a string directly or loaded via AJAX instead.

[property:boolean vertexTangents]Input Public

Defines whether precomputed vertex tangents, which must be provided in a vec4 "tangent" attribute, are used. When disabled, tangents are derived automatically. Using precomputed tangents will give more accurate normal map details in some cases, such as with mirrored UVs. Default is false.

[property:boolean wireframe]Input Public

Render geometry as wireframe. Default is *false* (i.e. render as flat polygons).

[property:string wireframeLinecap]Input Public

Define appearance of line ends. Possible values are "butt", "round" and "square". Default is 'round'.
This corresponds to the [link:https://developer.mozilla.org/en/docs/Web/API/CanvasRenderingContext2D/lineCap 2D Canvas lineCap] roperty and it is ignored by the [page:WebGLRenderer WebGL] renderer.

Notice - case insensitive.

[property:string wireframeLinejoin]Input Public

Define appearance of line joints. Possible values are "round", "bevel" and "miter". Default is 'round'.
This corresponds to the [link:https://developer.mozilla.org/en/docs/Web/API/CanvasRenderingContext2D/lineJoin 2D Canvas lineJoin] property and it is ignored by the [page:WebGLRenderer WebGL] renderer.

Notice - case insensitive.

[property:number wireframeLinewidth]Input Public

Controls wireframe thickness. Default is 1.
Due to limitations of the [link:https://www.khronos.org/registry/OpenGL/specs/gl/glspec46.core.pdf OpenGL Core Profile] with the [page:WebGLRenderer WebGL] renderer on most platforms linewidth will always be 1 regardless of the set value.
