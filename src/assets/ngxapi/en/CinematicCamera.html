CinematicCamera

Cinematic camera

Constructor

[constructor:CinematicCamera CinematicCamera]( [param:number fov]?, [param:number aspect]?, [param:number near]?, [param:number far]? )Inherited

Parameters

[parameter:number fov] - (optional) Camera frustum vertical field of view.
[parameter:number aspect] - (optional) Camera frustum aspect ratio.
[parameter:number near] - (optional) Camera frustum near plane.

[property:Matrix4 matrixWorld]Inherited

The global transform of the object. If the Object3D has no parent, then it's identical to the local transform .matrix.

[property:Matrix4 matrixWorldInverse]Inherited

This is the inverse of matrixWorld. MatrixWorld contains the Matrix which has the world transform of the Camera.

[property:boolean matrixWorldNeedsUpdate]Inherited

[property:Matrix4 modelViewMatrix]Inherited

This is passed to the shader and used to calculate the position of the object.

[property:string name]Inherited

Optional name of the object (doesn't need to be unique). Default is an empty string.

[property:number near]Inherited

Camera frustum near plane. Default is 0.1.
The valid range is greater than 0 and less than the current value of the .far plane.
Note that, unlike for the [page:OrthographicCamera OrthographicCamera], 0 is <em>not</em> a valid value for a PerspectiveCamera's near plane.

[property:Matrix3 normalMatrix]Inherited

This is passed to the shader and used to calculate lighting for the object. It is the transpose of the inverse of the upper left 3x3 sub-matrix of this object's modelViewMatrix.
The reason for this special matrix is that simply using the modelViewMatrix could result in a non-unit length of normals (on scaling) or in a non-perpendicular direction (on non-uniform scaling).
On the other hand the translation part of the modelViewMatrix is not relevant for the calculation of normals. Thus a Matrix3 is sufficient.

[method:void setViewOffset]( [param:number fullWidth], [param:number fullHeight], [param:number x], [param:number y], [param:number width], [param:number height] )Inherited

Parameters

[parameter:number fullWidth] - full width of multiview setup
[parameter:number fullHeight] - full height of multiview setup
[parameter:number x] - horizontal offset of subcamera
[parameter:number y] - vertical offset of subcamera
[parameter:number width] - width of subcamera
[parameter:number height] - height of subcamera

Sets an offset in a larger frustum. This is useful for multi-window or multi-monitor/multi-machine setups.
For example, if you have 3x2 monitors and each monitor is 1920x1080 and the monitors are in grid like this:
+---+---+---+
| A | B | C |
+---+---+---+
| D | E | F |
+---+---+---+

then for each monitor you would call it like this:

const w = 1920;
const h = 1080;
const fullWidth = w * 3;
const fullHeight = h * 2;

// A
camera.setViewOffset( fullWidth, fullHeight, w * 0, h * 0, w, h ) : this;
// B
camera.setViewOffset( fullWidth, fullHeight, w * 1, h * 0, w, h ) : this;
// C
camera.setViewOffset( fullWidth, fullHeight, w * 2, h * 0, w, h ) : this;
// D
camera.setViewOffset( fullWidth, fullHeight, w * 0, h * 1, w, h ) : this;
// E
camera.setViewOffset( fullWidth, fullHeight, w * 1, h * 1, w, h ) : this;
// F
camera.setViewOffset( fullWidth, fullHeight, w * 2, h * 1, w, h ) : this;
Note there is no reason monitors have to be the same size or in a grid.
